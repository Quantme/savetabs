When storing a tab with an unknown protocol...

Some protocols cannot be restored from saved tabs due to browser restrictions. This includes chrome:, data:, file:, javascript: and some about: URLs. These URLs are never stored.
This setting controls what the addon will do when it encounters a URL for which it does not know wether it can be restored or not. You can choose to store the tab anyways (but it could be that it is not restored) or you can choose to skip the tab while saving (and thereby saving some storage space). You only need this setting when you have installed an extension which adds support for a new protocol.
Hint: If you do not understand the previous paragraphs, just keep the setting on its default value (skip).
